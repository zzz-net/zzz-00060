# 抄表异常复核台 — 使用说明

## 启动

```
npm run dev
```

打开 http://localhost:5173

## 1. 批次导入

**操作方式：**
1. 左侧导航点「批次导入」
2. 点击虚线框内「选择 CSV 文件」，或直接把 CSV 拖到虚线区域
3. 文件名显示后点「开始导入」
4. 导入完成弹出结果窗口，显示：有效行数、错误行（含具体行号和原因）、生成异常数

**CSV 格式要求（UTF-8，含表头）：**
```
meterNo,meterName,prevReading,currReading,usage,readDate
M001,张三,100,200,100,2026-06-01
M002,李四,50,500,450,2026-06-01
```

**结果：**
- 表头字段不区分大小写，缺少列允许为空
- 某行 `currReading` 损坏（非数字）—— 报出具体行号和原因，其他有效行照常入库，不影响整批
- 同一文件内容重复导入—— 返回「该文件内容已导入过，不能生成重复异常」
- 下方「批次历史」表格展示所有已导入批次

## 2. 规则配置

**操作方式：**
1. 左侧导航点「规则配置」
2. 卡片上可直接点右侧开关启停规则，或点编辑图标修改
3. 右上「新增规则」按钮打开右侧面板

**支持的规则类型：**
| 类型 | 含义 | 可调参数 |
|---|---|---|
| 读数突增 spike | 当期用量 > 上期用量 × multiplier | multiplier（倍数） |
| 读数为负 negative | currReading < 0 | 无 |
| 读数回退 rollback | currReading < prevReading | 无 |
| 用量超限 overlimit | usage > limit | limit（阈值） |
| 空值检测 null_value | currReading 为空或无法解析 | 无 |

**结果：**
- 每次编辑规则自动生成新版本号，历史版本在「参数历史」里展开查看
- 停用的规则不会参与后续批次的异常识别
- 已产生的异常不受规则变更影响（每条异常记录当时的 ruleId 和 ruleVersion）

## 3. 异常复核

**操作方式：**
1. 左侧导航点「异常复核」
2. 顶部筛选栏：按批次、规则、状态（待复核/已改判/误报/已关闭）、表号关键字过滤，点「查询」
3. 待复核异常点右侧「改判」，打开右侧改判面板：
   - 判定结果：确认异常 / 误报
   - **变更异常类别（可选）**：下拉选目标规则，不选则保持原类别
   - 原因（必填建议）：说明改判依据
   - 备注（可选）：补充信息
   - 点「提交改判」
4. 已改判（confirmed/false_positive）的异常可点「关闭」
5. 已关闭的异常可点「重开」，状态回退到关闭前的状态

**结果：**
- 选择了变更类别：异常的异常类型、规则名、描述会更新为目标规则
- 不改类别：只更新状态
- 表格「改判信息」列实时显示：改判结果 + 换行显示原因 + 换行显示备注
- 每次改判/关闭/重开都在 `judgments` 表留痕，含时间戳、操作人、prevStatus→newStatus、prevRuleId→newRuleId

## 4. 复核报告

**操作方式：**
1. 左侧导航点「复核报告」
2. 查看 4 张统计卡片：总异常数、待复核、已改判、已关闭
3. 查看两张图表：异常类型分布饼图、批次异常分布柱图
4. 下拉选 CSV 或 JSON，点「导出报告」下载文件

**导出内容：**

**CSV 字段（含 BOM，Excel 直接打开不乱码）：**
异常ID、批次号、批次文件、表号、表名、上期读数、当期读数、用量、抄表日期、异常类型、规则名称、异常描述、状态、创建时间、**最新改判结果**、**改判原因**、**改判备注**、**改判操作人**、**改判时间**、**判定历史**（每次判定用 `; ` 分隔，各次之间用 ` || ` 分隔，包含状态流转、原因、备注、类别变更）

**JSON 格式：**
```json
{
  "success": true,
  "data": [
    {
      "id": "...",
      "anomalyType": "spike",
      "ruleName": "读数突增",
      "status": "confirmed",
      "judgments": [
        {
          "result": "confirm",
          "prevStatus": "pending",
          "newStatus": "confirmed",
          "reason": "...",
          "note": "...",
          "prevRuleId": "r3",
          "newRuleId": "r1",
          "operator": "复核员",
          "createdAt": "2026-06-17 ..."
        }
      ]
    }
  ]
}
```

## 5. 本地持久化与续跑

- 数据库文件：`data/meter-review.db`（SQLite）
- 程序关闭/重启后，以下内容完全保留且一致：
  - 所有批次、读数、异常记录
  - 所有规则版本
  - 所有改判/关闭/重开的判定历史、原因、备注
  - 复核报告统计口径（按类型/按批次分组）
- 清理测试数据：停止服务后删除 `data/meter-review.db`，重启即自动重建含默认规则的全新库

## 6. 回归测试

